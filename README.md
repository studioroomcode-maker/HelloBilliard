# Hello Billiard — 당구 AI 코치

4구 경로 시뮬레이터 + 3구(쓰리쿠션) 시스템 계산기. 공을 배치하거나 **당구대 사진을 찍으면** 물리 시뮬레이션으로 득점 경로를 성공률과 함께 제안합니다.

**라이브(PWA)**: https://hello-billiard.vercel.app — 폰 브라우저에서 열고 "홈 화면에 추가"하면 오프라인 동작하는 설치형 앱이 됩니다.

## 주요 기능

- **4구 경로 제안** — 두께×당점 전수 시뮬레이션으로 경로 3개 추천, 몬테카를로 30회 실측 성공률·공 모임도(%) 표시, ▶ 재생으로 실제 궤적 애니메이션
- **뱅크샷 / 쿠션 수 규칙** — 빈쿠션 경로 탐색, 추천/1쿠션/2쿠션/3쿠션↑ 규칙 선택 (규칙까지 성공률 판정에 반영)
- **실력 3단계** — 초급(중심 반 팁 이내 당점·두꺼운 두께만·큰 실행 오차)/중급/상급
- **📷 사진 인식** — 경기면 모서리 자동 감지 → 호모그래피 원근 보정 → 카메라 위치 역산으로 쿠션 벽(37mm)·공(33mm) 높이 시차 보정 → 공 색 자동 분류. 전 과정 온디바이스(사진 전송 없음)
- **3구 시스템 계산기** — 파이브앤하프(코너-50)·플러스·볼 시스템 + 물리 경로 제안(쓰리쿠션 규칙)
- **테이블 캘리브레이션** — 쿠션/구름 보정값을 기기에 저장해 내 구장에 맞춤

## 물리 엔진 (v4)

실측 비율(공 65.5mm / 대대 2844mm) 시간스텝 시뮬레이터 (DT 2.5ms):

- 미끄럼(μs=0.20)→구름(μr=0.012) 마찰 전이 — 끌기/밀기 곡선이 규칙이 아닌 물리로 발생
- 쿠션: 법선 반발 + 접선 마찰, 시네루 보정(회전 비율 연동 접선 상한 2/7→0.5, 스핀 홀드 0.25) — **파이브앤하프 차트로 캘리브레이션**
- 공-공: 반발 0.93 + 스로우 μ=0.06
- 검증 앵커: 알시아토레 실측(구름 1/2두께 분리각 33.7°, 스턴 탄젠트라인 등) + 파이브앤하프 교차검증 — `tests/validate-path.js`
- 알려진 한계: 깊은 뱅크(1쿠션수 30↑)는 실제보다 길게 시뮬됨(단일 임펄스 모델 한계) — 해당 구간은 차트 기반 계산기가 담당

## 프로젝트 구조

```
HelloBilli.html      앱 전체 (단일 파일: 스타일 + 4구/3구 엔진 + 비전 + UI)
index.html           루트 → 앱 리다이렉트 (+ 공유용 OG 태그 사본)
manifest.webmanifest PWA 매니페스트
sw.js                서비스워커 (오프라인 캐시) — 배포 시 VERSION 올리기
vercel.json          보안 헤더(CSP 등) + 정적 캐시 정책
robots.txt           크롤러 정책 + sitemap 위치
sitemap.xml          색인 대상 URL (HelloBilli.html)
og-image.png         공유 미리보기 1200×630 — 재생성: python tools/gen-og.py
icons/               앱 아이콘 (tests 아님 — 재생성: 세션 스크립트 gen-icons.js)
tools/               개발용 스크립트 (배포 제외)
tests/               검증 스위트 14종 + 실환경 사진 픽스처(gzip)
.github/workflows/   CI — push/PR마다 tests/run-all.js
.vercelignore        tests/ tools/ .github/ README 를 배포에서 제외
```

**메타 태그를 고칠 땐 `HelloBilli.html` 과 `index.html` 을 함께 고칠 것** — 공유되는 URL 은
루트(`/`)이고 스크래퍼가 meta refresh 를 따라가지 않을 수 있어 OG 태그를 양쪽에 둔다.

**CSP 주의** — `sw.js` 는 자기 응답 헤더의 CSP 를 상속한다. 구글 폰트 런타임 캐시가
`fetch()` 를 쓰므로 `connect-src` 에 `fonts.googleapis.com`·`fonts.gstatic.com` 이 필요하다.

## 테스트

```bash
node tests/run-all.js        # 전체 (~30초, 의존성 없음)
node tests/validate-path.js  # 물리 앵커만
```

- **엄격 테스트**(exit code): 물리 앵커, 사진 인식 합성 5시나리오, 실환경 필드 사진 2종
- **관대 테스트**(마커): 몬테카를로 분산이 있는 통합 테스트는 카드 개수 대신 단계 도달을 검증
- 물리 코드는 4구/3구 엔진이 2벌 존재하며 `validate-path.js`가 텍스트 동일성을 검사함 — **한쪽만 수정하면 CI가 실패**

## 배포

`main` push → Vercel 자동 배포. 큰 변경 시 `sw.js`의 `VERSION`(`hb-vN`)을 올려 구 캐시를 정리할 것.
